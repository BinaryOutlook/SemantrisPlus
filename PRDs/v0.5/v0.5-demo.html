Architecture extraction map

Pull responsibilities out of the two crowded command posts.

app.py Keep route entry clear. Delegate vocabulary, session state, serializers, run lifecycle, and Blocks orchestration.
Gameplay modules Keep mode rules separate. Strengthen contracts without forcing one generic state model.
Persistence Keep run-store behavior optional, local-first, and explicit in serialized payloads.
llm_client.py Turn into a smaller facade while semantic-engine modules own contracts, prompts, parsers, providers, fallback, and factory logic.
frontend/src Share API, timer, modal, and persistence helpers while preserving mode-specific controller readability.
templates Extract repeated head, theme, topbar, and modal markup only where it makes pages easier to read.

Execution phases

Advance by checkpoint, not by hope.

Phase 1

Decompose backend command-post responsibilities while preserving routes.

Phase 2

Split semantic engine responsibilities behind a compatibility facade.

Phase 3

Strengthen backend state contracts and persistence behavior tests.

Phase 4

Consolidate frontend API, HUD, modal, timer, and template infrastructure.

Phase 5

Add CI, dependency hygiene notes, README updates, and final PRD change log.

Definition of ready

The game stays playable after every checkpoint.

Required closeout: backend tests, frontend build, typecheck, Vitest, Biome, and Playwright all pass. Documentation reflects any durable ownership changes.

Primary hazard

Do not send multiple agents into the same crowded file.

The Theater Commander must sequence edits to `app.py` and `llm_client.py`. Parallelism begins after ownership boundaries are extracted and tests are green.

v0.5 succeeds when Semantris Plus can receive five coordinated agents without losing route stability, ranking safety, frontend clarity, or documentation discipline.
