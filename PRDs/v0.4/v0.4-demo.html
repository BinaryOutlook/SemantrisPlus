Semantris Plus v0.4

Reliability, semantic quality, and product readiness without a rewrite.

This packet frames v0.4 as a strengthening pass: keep Flask, Jinja, and TypeScript, but add a better semantic engine, deterministic end-to-end confidence, typed runtime settings, and a local persistence spine for run history and best scores.

Keep Flask + Jinja + TS Improve ranking quality Add Playwright Add typed settings Add SQLAlchemy persistence Evaluate Vite carefully

Pillar 1: Semantic Engine

  • Add request-level caching for repeat ranking operations.
  • Strengthen local fallback quality beyond the current heuristic-only path.
  • Keep Gemini and OpenAI-compatible routing isolated behind the ranking layer.

Pillar 2: Confidence and Tooling

  • Add deterministic Playwright flows for all three modes.
  • Add typed runtime settings and clearer startup validation.
  • Add frontend linting and only migrate to Vite if the win is real.

Pillar 3: Persistence

  • Introduce SQLAlchemy with SQLite by default.
  • Store completed runs, best-score metadata, and mode-pack summaries.
  • Expose just enough saved-state information to enrich the game-over flow.

Pillar 4: AI-Ready Execution

  • Spell out file ownership instead of relying on implied structure.
  • Keep implementation phases small and testable.
  • Make the packet readable enough that future agents can code from it directly.
Phase 1

Typed Settings and Fake Ranker

Phase 2

Semantic Cache and Fallback

Add cache abstractions, improve remote-failure behavior, and keep all ranking validation discipline intact.

Persistence and Best Scores

Save completed runs locally, compute best scores, and surface that information in a small, explicit product-friendly way.
